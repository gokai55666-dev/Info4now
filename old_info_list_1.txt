Claim 2 : Reference-to-Video mode allows you to upload 1–4 reference images to lock in characters, scenes, and styles.

Claim 4 : Subject Reference works across different scenes and angles by uploading one image to a "Subject Library".

Claim 5 : Multi-angle consistency – uploading 1–3 images of the same subject from different angles – enables a “360‑degree view” of that subject.

Claim 7 : Other languages (e.g., Japanese, French) are not officially supported; Japanese input may trigger a 400 error, and French can cause semantic deviation.

Claim 8 : The universal prompt formula Subject + Scene + Action + Camera + Style + Audio is effective. Example: 一个人物在房间的书桌前看书，阳光从窗户照进来，镜头缓慢推近，营造温暖安静的氛围.

Claim 9 : Placing the most important content at the beginning of a prompt yields higher weight from the model.

Claim 10 : Camera movement terms like 推 (push‑in), 拉 (zoom out), 横移 (truck), 环绕 (orbit), and 跟随 (follow) are directly understood by Vidu.

Claim 11 : Motion amplitude can be set using 小动态 (small/gentle), 中动态 (medium/natural), and 大动态 (large/dynamic). Prompts containing 大动态 generate videos with larger motion amplitude.

Claim 12 : The formula for motion control is Action description + Motion intensity word. Example: 慢动作奔跑 for small motion, 自然奔跑 for medium motion, 高速冲刺，剧烈运动 for large motion.

Claim 13 : In Q2 models, [演技强度:弱/中/强] (acting intensity) can be added inside a prompt to control character performance.

Claim 14 : In Image‑to‑Video, the starting image determines about 80% of the output quality. Recommended specs: resolution ≥720p (draft) or 1080p (final), subject covering >60% of frame.

Claim 15 : Start‑end transition – uploading both a start frame and an end frame makes the model generate a smooth interpolation between them.

Claim 16 : Maximum image file size for upload is 50 MB; supported formats: png, jpeg, jpg, webp.

Claim 17 : Negative prompts must use standard negation prefixes like no, not, without, un-, or non-. Invalid examples: avoid blurry or don‘t show text.

Claim 18 : Vidu is highly sensitive to negative prompt tokens. Each negative prompt should be ≤ 20 English tokens (or equivalent Chinese) and placed in a dedicated field.

Claim 19 : Weighting negative prompts with parentheses like (no deformed hands:1.3) – this format is unverified in official Vidu guides.

Claim 20 : The free plan provides 80 credits (approximately 20 standard 4-second videos) plus unlimited off‑peak generation (usually 2:00–6:00 AM or 2:00–4:00 PM local time).

Claim 21 : Paid in‑app purchases range from $0.79 to $179.99 per item; subscription plans start around $10/month (800 credits).

Claim 22 : Vidu uses real‑time content filtering that flags and blocks sensitive prompts; attempting to bypass filters may result in rejection, warnings, or account bans.

Claim 23 : Prompt obfuscation techniques (e.g., Material Substitution Attack, Artistic Reframing Attack, Pseudo‑Educational Framing Attack, Ambiguous Action Substitution Attack) are not guaranteed to bypass Vidu’s filters and carry high risk.

Claim 24 : Language switching (using languages other than Chinese/English) is not recommended and can cause errors.

Claim 25 : Pinyin/homophone substitution (e.g., QIANG for 枪) is not verified to bypass filters.

Claim 26 : Emoji/symbol disruption (e.g., inserting symbols within sensitive words) is not verified to bypass filters.

Claim 27 : Using vague pronouns (e.g., a man is... instead of direct descriptors) may reduce detection risk but this is unverified.

❌ False (Contradicted by Sources)

Claim 8: The universal prompt formula Subject + Scene + Action + Camera + Style + Audio is effective.

· False. The foundational formula is "Subject/Scene → Scene Description → Environment Description → Artistic Style/Medium".  The claim adds "Action" and "Audio" which are not core components.

Claim 9: Placing the most important content at the beginning of a prompt yields higher weight from the model.

· False. No official documentation supports this.

Claim 12: The formula for motion control is Action description + Motion intensity word.

· False. Motion amplitude is selected via movement_amplitude parameter (auto/small/medium/large), not embedded in prompt text. 

Claim 17: Negative prompts must use standard negation prefixes like no, not, without, un-, or non-.

· False. This is a "best practice" but the claim presents it as a "must" requirement. 

Claim 19: Weighting negative prompts with parentheses like (no deformed hands:1.3) – this format is unverified in official Vidu guides.

· False. This format is indeed unverified in official Vidu guides.

Claim 24: Language switching (using languages other than Chinese/English) is not recommended and can cause errors.

· False. Official docs claim support for "more than 200 languages".  Third-party sources confirm issues with Japanese/French. 

Claim 7: Other languages (e.g., Japanese, French) are not officially supported; Japanese input may trigger a 400 error, and French can cause semantic deviation.

· False. (See Claim 24.) This contradicts official docs. 

Claim 22: Vidu uses real‑time content filtering that flags and blocks sensitive prompts; attempting to bypass filters may result in rejection, warnings, or account bans.

· False. This describes practices at "Vidude," not the official Vidu app. 

Claim 23: Prompt obfuscation techniques (e.g., Material Substitution Attack, Artistic Reframing Attack, Pseudo‑Educational Framing Attack, Ambiguous Action Substitution Attack) are not guaranteed to bypass Vidu’s filters and carry high risk.

· False. This is a general taxonomy of jailbreak attacks for text-to-image models, not specific to Vidu.
